Sesión 2


Hyper Text Markup Languaje

Lectura no secuencial

Diferentes vinculos y redes de informacion conectadas entre si.

Una etiqueta puede contener a otra etiqueta.

Identación


Aplicar sucesivos niveles de márgenes de acuerdo a la anidación de etiquetas.
Permite identificar con mayor facilidad problemas de anidación o desbalance en etiquetas de cierre.
Se puede indentar con tabuladores o espacios.

Recomendaciones de escritura

  • Cuida siempre la indentación
  • Prefiere minúsculas para etiquetas y atributos
  • Prefiere comillas dobles para atributos
  • Evita generar líneas demasiado largas
  • Elimina espacios blancos innecesarios

Recomendaciones para nombres de archivos y carpetas

  • Prefiere minúsculas
  • Utiliza sólo letras, números y guiones. Evita usar caracteres inusuales, tildes, símbolos, comillas, etc
  • Utiliza guiones para separar palabras. Evita siempre usar espacios
  • Definir nombres cortos y relevantes, junto con una convención de nombres

Ventajas de una buena semántica


Mejora la accesibilidad del contenido para usuarios que utilizan lectores de pantalla
Puede ser considerado por los motores de búsqueda para influir en el ranking de una página (SEO)
Permite recorrer y rastrear bloques de código significativo
Sugiere el tipo de datos con que debe ser completado

Flexbox


Entrega a un contenedor la habilidad para alterar el ancho o alto, y espaciados de sus elementos para ajustarse lo mejor posible al espacio disponible.


Un contenedor flexible expande sus elementos para rellenar el espacio libre, o los comprime para evitar que rebasen el área prevista.


Los contenedores flexibles funcionan en un eje, horizontal o vertical



  • Contenedor flexible: Elemento con propiedad display: flex;
  • Elementos flexibles: Descendientes directos de un contenedor flexible
  • Eje principal: Sobre el cual se suceden los elementos flexibles
  • Eje transversal: Perpendicular al eje principal
  • Líneas : Los elementos flexibles pueden quedar contenidos en una línea o utilizar varias según su ancho

Media queries


Las media queries o consultas de medios son una forma de aplicar estilos en función del tipo de dispositivo o de aspectos específicos.


/body { background-color: #fff;
}
/* Mínimo ancho de pantalla de 768px */
@media (min-width: 768px) {
body {
background-color: purple;
}
}
/* Vista previa de impresión e impresos */
@media print {
.menu {
display: none;
}
}/

Uso de imágenes


Formatos comunes para web

png → Imagen de trama, para colores uniformes
gif → Intercambio de imágenes, sólo 256 colores
svg → Imagen vectorial escalable, código

Optimización de imágenes

-Escoger el formato de imagen correcto.
-Utilizar imágenes en un tamaño similar al que será mostrará en pantalla.
-Comprimir imágenes https://compressor.io/.
-Preferir webp en vez de jpg.

Imágenes adaptables

Entregar un archivo de imagen diferente dependiendo de características del dispositivo, utilizando los atributos srcset y sizes

Se entrega una imagen distinta dependiendo del ancho de pantalla: img srcset="elva-fairy-320w.jpg 320w,
elva-fairy-480w.jpg 480w,
elva-fairy-800w.jpg 800w"
sizes="(max-width: 320px) 280px,
(max-width: 480px) 440px,
800px"
src="elva-fairy-800w.jpg" alt="Elva dressed as a fairy">

Se entrega una imagen distinta dependiendo del ancho de pantalla:

img srcset="elva-fairy-320w.jpg,
elva-fairy-480w.jpg 1.5x,
elva-fairy-640w.jpg 2x"
src="elva-fairy-640w.jpg" alt="Elva dressed as a fairy">


PPI ~ PPP Pixeles por pulgada

Densidad de pixeles de una imagen, que relaciona el número total de pixeles de una pantalla con su tamaño en pulgadas


Accesibilidad


Discapacidades:
  • Auditiva
  • Cognitiva
  • Neurológica
  • Física
  • Habla
  • Visual

Situaciones:
  • Distintos dispositivos de entrada
  • Adultos mayores
  • Discapacidades temporales
  • Limitaciones situacionales
  • Características técnicas

¿Porqué es importante la accesibilidad?

-Acceso a la información como derecho humano
-Igualdad de acceso e igualdad de oportunidades para personas con distintas capacidades
-Inclusión social: adultos mayores, población en áreas rurales, en países en vías de desarrollo...
-Puede ser un requerimiento legal en ciertos contextos
-Razones de negocio: abarcar un mayor mercado, cumplimiento de responsabilidad pública y políticas de inclusión


Principios de accesibilidad

Perceptible La información y los componentes de la interfaz de usuario deben ser presentados a los usuarios de modo que ellos puedan percibirlos.
Operable Los componentes de la interfaz de usuario y la navegación deben ser operables.
Comprensible La información y el manejo de la interfaz de usuario deben ser comprensibles.
Robusto El contenido debe ser suficientemente robusto como para ser interpretado de forma fiable por una amplia variedad de aplicaciones de usuario, incluyendo las ayudas técnicas.

-Metadatos-


Permiten al buscador tener una perspectiva general del contenido de la página y al autor del contenido de posicionar sus palabras claves


Técnicamente no tiene limitación, pero se recomienda que tenga como máximo 70 a 75 caracteres ya que sobre esa cantidad el buscador la corta automáticamente.
Meta description
Se recomienda que tenga como máximo 160 caracteres; se inserta en la cabecera del documento.
meta name="description" content="Descripción del contenido de la página">
La etiqueta meta “keywords” está obsoleta y no se debe utilizar


-Atributo “alt” en imágenes-


El atributo “alt” especifica un texto alternativo para una imagen. Es fundamental para la accesibilidad de personas ciegas, pero además mejora la indexación de los contenidos de una página al otorgar contexto a las imágenes utilizadas.
img src=”foto.jpg” alt=”Domos el LLano (Putaendo)”


-Sitemaps.xml-


Los sitemaps son archivos en los que se proporciona información sobre las páginas, los vídeos y otros archivos de tu sitio, así como sobre las relaciones que hay entre ellos.


-URL Canónica-


Un contenido puede ser accesible a través de varias URLs, pero sólo una es la “URL canónica”.
Siempre es una URL absoluta.
La definición de una URL canónica evita el problema del contenido duplicado.
Se define mediante un metadato en la sección del documento HTML.
link rel="canonical" href="https://www.ead.pucv.cl/">


-Otros factores relevantes para posicionamiento-


Optimización para móviles La usabilidad del sitio en un dispositivo móvil influye positivamente para su posicionamiento. Google está migrando a indexación “mobile-first” en la mayoría de los sitios.
Performance
La velocidad de carga, tanto en móviles como en escritorio, influye potentemente en el posicionamiento de un sitio.
Conexión sobre HTTPS
El funcionamiento sobre conexiones seguras se considera como una señal para el posicionamiento.
Datos estructurados
La presencia de microdatos y microformatos permiten una mejor indexación de los contenidos de un sitio
